Hard requirements:
- shots must come from the shots inside the block-first structure. shotNumber must be continuous. Do not skip, duplicate, or reorder shots.
- Every shot must include the seven minimum fields: shotNumber, durationSec, visualAction, charactersAndScene, camera, videoPrompt, sound.
- durationSec must be an integer from 1 to 5. A single shot must never exceed 5 seconds.
- Top-level durationSec must equal the sum of all shot durationSec values and be close to the user request.
- You must output videoBlocks. videoBlocks is the downstream video generation arrangement, not a transition table.
- videoBlocks must cover every shot exactly once in original shotNumber order. Do not skip, duplicate, or reorder shots.
- Every videoBlock must include prompt. prompt is the final text sent to the video model, not an internal summary.
- single must contain exactly 1 shot.
- A group must contain 2-9 adjacent shots, and the summed durationSec of those shots must be 2-15 seconds.
- Use gridMode "2x2" for 2-4 adjacent shots; use gridMode "3x3" for 5-9 adjacent shots.
- Do not group only to fill the grid; empty cells are allowed.
- Prefer group by default. Use single only for severe space/time jumps, static display shots, or cases where continuous generation would contaminate the image.
- If adjacent shots are suitable for grouping but exceed 15 seconds total, split within the 15-second limit.

videoBlocks[].prompt writing rules:
- For type single, prompt describes only that final single-shot video: subject, scene, action, camera movement, lighting, style, aspect ratio, and sound rhythm. Do not mention grids, arrangements, shotNumber, or system fields.
- For type group, prompt must be rewritten as one complete continuous video segment. Do not mechanically concatenate shots[].videoPrompt.
- A group prompt must begin with overall genre, cinematography, color, lighting, movement rhythm, subject continuity, and aspect ratio.
- A group prompt must preserve the timeline and write each segment as [00:00-00:03].
- Each group prompt segment must describe image content, subject action, camera position/movement, sound rhythm, and continuity or transition logic with adjacent shots.
- A group prompt must state that the reference image/grid is only a storyboard index and the output must be full-screen continuous video. Never generate a 2x2 grid, 3x3 grid, collage, split screen, shot numbers, grid labels, or UI.
- A group prompt must ignore Empty placeholder cells. Do not generate black screens, white screens, or placeholder frames.
