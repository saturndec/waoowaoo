# 修复个人中心宽度、OpenAI 兼容模型自动获取与分集轮询卡死

## Goal
修复用户在个人中心遇到的 3 个问题：内容区域过窄、OpenAI 兼容层不自动拉取模型、智能分集轮询卡死导致持续请求。

## Requirements
- 个人中心右侧内容区支持全宽展示，不再限制为窄列。
- OpenAI 兼容层在配置 `Base URL + API Key` 后可自动获取并展示模型列表。
- 智能分集任务在长期 `queued` / 超时场景下应停止无限轮询并给出明确失败反馈。

## Acceptance Criteria
- [ ] API 配置页面在 2K 屏下内容区不再被 `max-width` 限制。
- [ ] 新增 OpenAI 兼容层后，模型列表可自动出现（至少包含文本模型）。
- [ ] 分集分析不再无限请求 `/api/tasks/:id`，超时或长时间排队会结束并抛出可读错误。
- [ ] 对任务轮询新增回归测试，覆盖“排队超时停止轮询”场景。

## Technical Notes
- 变更范围为 `profile` 前端布局、`api-config` 前后端、`task/client` 轮询逻辑。
- 遵循“显式失败、零隐式回退”，不做静默兜底。
