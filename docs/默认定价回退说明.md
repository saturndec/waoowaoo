# 默认定价回退说明

## 变更目标
- 当模型在内置价格目录中不存在，或能力组合未命中定价 tier 时，不再直接报错。
- 改为使用系统默认价格，避免保存配置和计费链路被硬阻断。

## 变更范围
- 新增默认定价模块：`src/lib/model-pricing/defaults.ts`
- 调整 API 配置保存校验：`src/app/api/user/api-config/route.ts`
- 调整计费价格解析逻辑：`src/lib/billing/cost.ts`

## 默认价格策略
- 文本模型：
- 输入 token 单价默认 `0.2`
- 输出 token 单价默认 `0.6`
- 图像模型默认 `0.144`
- 视频模型默认 `0.2`
- 语音模型默认 `0.0144`
- 音色设计默认 `0.2`
- 口型同步默认 `0.5`

## 行为变化
- `PUT /api/user/api-config` 在 `BILLING_MODE != OFF` 下，默认模型若无内置定价不再返回 `DEFAULT_MODEL_PRICING_NOT_CONFIGURED`。
- 计费中心在以下场景使用默认价格：
- `not_configured`（模型未配置价格）
- `missing_capability_match`（能力组合未命中价格 tier）
- 原有 `ambiguous_model` 仍然保留显式报错，避免歧义模型静默计费。

## 使用说明
- 无需新增环境变量，默认价格已内置。
- 用户可继续保存 `provider::modelId` 格式的默认模型，即使该模型暂未进入内置定价目录。
