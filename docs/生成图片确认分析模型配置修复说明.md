# 生成图片确认分析模型配置修复说明

## 问题现象
- 在项目内执行“确认并生成图片”时失败，前端提示：
  - `确认失败: 请先在项目设置中配置分析模型`

## 根因分析
- 角色档案确认 worker 仅依赖项目表 `novelPromotionData.analysisModel`。
- 当项目级模型为空时，即使任务 payload 已携带 `analysisModel`（由任务路由注入），worker 仍会直接报错。

## 修复方案
- 统一模型解析顺序：
  - `payload.model`
  - `payload.analysisModel`
  - `project.novelPromotionData.analysisModel`
- 在 `resolveProjectModel` 中支持 fallback 模型，并返回最终可用的 `analysisModel`。
- 当三者都为空时，继续显式失败，不做隐式兜底。

- `src/lib/workers/handlers/character-profile.ts`
