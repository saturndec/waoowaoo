# 开始创作分析模型未配置修复说明

## 问题现象
- 点击“开始创作”后在准备阶段失败，报错：`analysisModel is not configured`。

## 根因分析
- `story-to-script` 与 `script-to-storyboard` worker 的模型解析顺序仅使用：
  - `payload.model`
  - `novelPromotionProject.analysisModel`
- 当项目级 `analysisModel` 为空，且请求体未显式传 `model` 时，即使任务已带 `payload.analysisModel`，worker 仍会判定未配置并失败。

## 修复方案
- 统一调整两个 worker 的模型解析优先级：
  - `payload.model`
  - `payload.analysisModel`
  - `novelPromotionProject.analysisModel`
- 保持无模型时显式失败，不引入静默降级。

## 变更文件
- `src/lib/workers/handlers/story-to-script.ts`
- `src/lib/workers/handlers/script-to-storyboard.ts`
- `tests/unit/worker/story-to-script.test.ts`
- `tests/unit/worker/script-to-storyboard.test.ts`

## 回归测试
- 新增用例（`it` 名称包含 bug 场景）：
  - `bugfix: falls back to payload.analysisModel when project analysisModel is empty`
- 验证点：
  - 当项目 `analysisModel` 为空且 `payload.analysisModel` 存在时，任务可继续执行；
  - 能力解析调用使用 `payload.analysisModel` 作为 `modelKey`。
