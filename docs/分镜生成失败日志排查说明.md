# 分镜生成失败日志排查说明

## 目标
- 定位「分镜下方生成失败」的具体失败环节，避免只看到前端通用报错而无法还原原因。

## 本次新增日志点

### 1. 分镜重生图 API 提交链路
- 文件：`src/app/api/novel-promotion/[projectId]/regenerate-panel-image/route.ts`
- 关键日志：
  - `regenerate panel image request received`
  - `regenerate panel image model config loaded`
  - `resolve panel image capability options failed`（错误码：`PANEL_IMAGE_CAPABILITY_RESOLVE_FAILED`）
  - `panel image capability options resolved`
  - `panel image task submitted`

### 2. 分镜 worker 生图链路
- 文件：`src/lib/workers/handlers/panel-image-task-handler.ts`
- 关键日志：
  - `panel reference image normalization started`
  - `panel reference image normalization issue`
  - `panel reference image normalization failed`（错误码：`PANEL_REFERENCE_IMAGE_NORMALIZATION_FAILED`）
  - `panel reference image normalization completed`
  - `panel image generation started`
  - `panel image prompt resolved`
  - `panel candidate generation started`
  - `panel candidate source generated`
  - `panel candidate generation failed`（错误码：`PANEL_CANDIDATE_GENERATION_FAILED`）
  - `panel candidate uploaded`
  - `panel candidate upload failed`（错误码：`PANEL_CANDIDATE_UPLOAD_FAILED`）
  - `panel image generation persisted`

### 3. 公共媒体拉取链路（/m 路由）
- 文件：`src/app/m/[publicId]/route.ts`
- 关键日志：
  - `public media upstream fetch started`
  - `public media upstream fetch failed`（错误码：`MEDIA_UPSTREAM_FETCH_FAILED`）
  - `public media upstream returned non-ok status`（错误码：`MEDIA_UPSTREAM_BAD_STATUS`）
  - `public media not found`
  - `public media storage key missing`

## 如何使用这些日志定位问题
- 第一步：看 API 是否提交成功  
  搜索 `api.novel-promotion.regenerate-panel-image`，确认是否出现 `panel image task submitted`。
- 第二步：看 worker 是否进入候选图生成  
  搜索 `worker.panel-image`，确认是否出现 `panel candidate generation started`。
- 第三步：定位失败阶段  
  - 若看到 `PANEL_REFERENCE_IMAGE_NORMALIZATION_FAILED`：参考图在归一化阶段全部失败，继续看 `normalizationIssues` 内每张图的 `code/stage/message/input`。
  - 若看到 `PANEL_IMAGE_CAPABILITY_RESOLVE_FAILED`：模型能力参数解析失败（例如缺少 resolution）。
  - 若看到 `PANEL_CANDIDATE_GENERATION_FAILED`：模型调用/网络失败。
  - 若看到 `PANEL_CANDIDATE_UPLOAD_FAILED`：上传到 COS 失败。
- 第四步：若前端报「不是有效图片」或 `/m/...` 500  
  搜索 `api.media.public`，查看是否是 `MEDIA_UPSTREAM_FETCH_FAILED`（如 ECONNREFUSED）或上游非 2xx。
